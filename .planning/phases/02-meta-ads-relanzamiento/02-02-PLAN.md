|----------|-------------|
| API → Meta Ads | Llamadas POST crean objetos reales con presupuesto — error en payload puede crear campañas duplicadas o con targeting incorrecto |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | URL de destino sin UTMs en ad creative | mitigate | Verificar en checkpoint humano que la "Website URL" del ad muestra los UTMs completos. Si Hotmart limpia UTMs: agregar parámetro `src=meta-ads` nativo de Hotmart como fallback de atribución. |
| T-02-06 | Denial of Service | Activación accidental de campaña sin revisión | mitigate | Checkpoint obligatorio antes de activar. Campaña creada en PAUSED. Activación solo por comando explícito post-aprobación. |
| T-02-07 | Information Disclosure | META_ACCESS_TOKEN hardcodeado en ad_ids documentados | mitigate | 02-ASSETS.md contiene IDs de objetos Meta (no credenciales). El token siempre se lee de .env via `source`. |
| T-02-08 | Tampering | Targeting incorrecto (CABA no excluida) | mitigate | Verificar en Meta Ads Manager Audience Definition que muestra "Argentina excluding Buenos Aires (50km radius)" antes de aprobar. Si la API no aplicó la exclusión, corregir manualmente en UI antes de activar. |
| T-02-09 | Elevation of Privilege | Token con más permisos de lo necesario activando campañas no deseadas | accept | Token tiene acceso a toda la ad account — riesgo inherente. Mitigación: Crear todo en PAUSED y activar solo después del checkpoint humano. Token expira ~10 mayo 2026. |
